**Q1：数据采集方式？**
──────────────────────────────────────────────────────────
**决策**：爬虫方案为主，同步申请阳光高考平台 API，两者并行推进。
**理由**：
  - 爬虫可立即启动（无需等待 API 审批），先爬 3 省试点验证可行性；
  - 阳光高考 API 数据更权威，若申请通过则优先使用 API 数据源；
  - 若 API 申请被拒（需教育机构资质），已有爬虫方案兜底。
**实施**：
  - 今日（2026-06-10）开始写山东/河南/广东 3 省爬虫；
  - 同步填写阳光高考 API 申请表（https://api.eol.cn/docs/），等待审批；
  - 若 2 周内 API 未批，则全面转爬虫方案。

**Q2：AI 模型用 Claude 还是国产？**
──────────────────────────────────────────────────────────
**决策**：MVP 阶段全用 Claude Sonnet 4.6；用户量上升后引入混合方案降成本。
**理由**：
  - Claude Sonnet 4.6 推理质量最高，适合高考志愿推荐等复杂场景；
  - 国产模型（DeepSeek-V3、GLM-4）成本低 10 倍，但推理质量略逊；
  - MVP 阶段用户少，Claude 成本可控（预计 <200 元/月）；
  - 用户量上来后（>1000 月活），再引入混合方案：简单场景用国产，复杂场景用 Claude。
**混合方案（未来）**：
  - 高考志愿推荐（复杂推理）→ Claude Sonnet 4.6（~0.015 元/千 token）
  - 考试错题分析（图片 OCR+分析）→ Claude Sonnet 4.6（~0.03 元/张）
  - 日常对话（简单问答）→ DeepSeek-V3 或 GLM-4（~0.001 元/千 token）
  - 数据批量处理（离线）→ DeepSeek-V3（~0.001 元/千 token）

**Q3：认证是否简化为"手机号+密码"，不发短信验证码？**
──────────────────────────────────────────────────────────
**决策**：是，MVP 阶段仅"手机号+密码"，省掉短信成本；v2.0 再加短信验证。
**理由**：
  - 短信验证码成本约 0.03-0.05 元/条，MVP 阶段无收入，需控制成本；
  - "手机号+密码"足够安全（加 bcrypt 加密 + 登录失败锁定）；
  - 短信验证留到 v2.0（有收入后再加），作为付费用户的"账号安全升级"。
**安全措施（MVP 阶段）**：
  - 密码必须 bcrypt 加密（cost=12），禁止明文存储；
  - 登录失败 5 次锁定 15 分钟（防暴力破解）；
  - HTTPS 必须（阿里云/腾讯云免费证书）；
  - JWT token 有效期 7 天，支持刷新。

**Q4：OCR 上传的错题图片是否 24 小时后自动删除？**
──────────────────────────────────────────────────────────
**决策**：OCR 完成后立即删除原图（不等 24 小时），只保留 OCR 提取的文字结果。
**理由**：
  - 高考考生多为未成年人，图片可能包含人脸/学校制服，隐私风险高；
  - 原图无业务价值（OCR 完成后只需文字结果）；
  - 保留原图增加数据泄露风险，不符合"最小数据留存"原则。
**实施**：
  - 用户上传图片 → OCR 识别 → 提取文字结果 → 立即删除原图文件；
  - 数据库只存文字结果（如："错题：三角函数，错误原因：公式记错"）；
  - 若用户坚持要存图片（如用于"错题本打印"），需明确授权（默认不存）；
  - 在《隐私政策》中明确说明："图片识别后即刻删除，不留存服务器"。

**Q5：是否统一把所有分数折算成 750 分制？**
──────────────────────────────────────────────────────────
**决策**：是，统一折算成 750 分制，同时保留原始分数（展示时同时显示）。
**理由**：
  - 各省总分不同（浙江 750、上海 660、海南 900），不折算无法跨省比较；
  - 折算公式：标准分 = 原始分 ÷ 原始总分 × 750；
  - 展示时同时显示"原始分/原始总分"和"标准分/750"，避免误导用户。
**数据库设计**：
  ```sql
  CREATE TABLE college_scores (
    ...
    original_score INTEGER,       -- 原始分数线（如浙大在浙江录取线 660）
    original_total INTEGER,       -- 原始总分（浙江 750、上海 660、海南 900）
    standard_score INTEGER,      -- 折算成 750 分制（方便比较）
    ...
  );
  ```
**示例**：
  - 浙江大学在浙江录取线 660 分（总分 750）→ 标准分 660（无需折算）
  - 浙江大学在上海录取线 440 分（总分 660）→ 标准分 = 440÷660×750 = 500
  - 浙江大学在海南录取线 600 分（总分 900）→ 标准分 = 600÷900×750 = 500

1. 项目概述 —— 愿景与MVP边界
********************************************************************************

1.0 一句话定义
──────────────────────────────────────────────────────────
人生规划系统是一个陪伴用户从高中到职场早期的 AI 数据伙伴。
MVP阶段（v1.0）聚焦：高中考试分析 → 高考志愿填报 → 大学规划 → 求职准备。
完整愿景覆盖童年到老年，但v1.0只做4个核心场景。

1.1 产品定位（MVP阶段，2026年）
──────────────────────────────────────────────────────────
  核心用户: 中国高中生（15-18岁）+ 家长 + 大学生（18-22岁）
  核心场景: 考试错题分析 → 志愿填报 → 大学规划 → 求职准备
  核心价值: 用数据+AI替代高价真人咨询（张雪峰5999元 vs 我们199元/年）
  
  与竞品差异化:
  - 夸克/掌上高考: 只有志愿推荐，没有长期数据积累
  - 张雪峰: 贵，无法规模化，不记录历史数据
  - 我们: 数据从高中开始积累，越用越准，价格是真人1/30

1.2 愿景层（10年目标，不作为当前开发范围）
──────────────────────────────────────────────────────────
  [保留v0.1.0的1.2-1.8章节作为愿景参考，但明确标注"非当前优先级"]
  
  愿景覆盖: 小学→高中→大学→研究生→职场→中年→晚年（全文保留在附录A）
  当前MVP只做: 高中考试分析 + 高考志愿 + 大学规划 + 求职准备

1.3 MVP功能边界（v1.0，2026年Q3发布）
────────────────────────────────
  ✅ 必须做（P0）:
    1. 考试记录 + 逐题错题分析（OCR拍照录入）
    2. 高考志愿填报AI推荐（冲稳保三档）
    3. 日常活动记录 + AI反馈
    4. 人生目标设定 + 进度追踪
    5. AI教练对话（高考/学习/职业三个领域）
  
  🔲 暂不做（P1，v2.0）:
    - 大学课程规划、实习管理
    - 考研规划
    - 语音输入
    - 可穿戴设备同步
    - 社交网络分析
  
  ❌ 明确不做（当前版本）:
    - 小学/初中场景
    - 心理健康诊断（需医疗资质）
    - 金融投资建议（需金融牌照）
    - 中年/晚年场景

2.9 数据获取法律风险评估 ★新增
──────────────────────────────────────────────────────────

  核心原则: 不做违法的事，但"可能违反服务条款（ToS）"≠"违法"。
  以下按风险等级分类，优先做低风险的数据源。

2.9.1 低风险（可直接做，无法律隐患）
──────────────────────────────────────────────────────────
  数据源                    法律依据              风险说明
  ─────────────────────────────────────────────────────────────────
  阳光高考平台(yxzy.cn)     教育部官方公开数据    无风险。教育部主导的公共数据平台
  省考试院官网(公开页面)    政府信息公开条例     无风险。政府公开信息不受版权保护
  教育部课标/考纲           官方公开文件          无风险。官方文件不受版权限制
  高校就业质量报告          《高等学校信息公开办法》 无风险。教育部要求高校公开的信息
  软科/QS排名               公开商业报告         低风险。仅为引用公开数据
  研招网(yz.chsi.com.cn)   教育部公开数据        无风险。官方公开数据平台

2.9.2 中风险（可做，但需注意使用方式）
──────────────────────────────────────────────────────────
  数据源                    法律依据              风险说明
  ─────────────────────────────────────────────────────────────────
  院校招生章程              高校官网公开信息     低风险。但需注意各校招生章程的
                                                 版权声明（有些声明"禁止转载"）
  题库网站(学科网/菁优网)   版权法               高风险。商业题库受著作权保护。
                                                 策略: 不爬取，只考虑合规合作

  降低中风险的具体措施:
    1. 只爬取"必须公开"的数据（政府要求的信息公开）
    2. 爬取频率降低到合理范围（不造成对方服务器压力）
    3. 数据使用时标注来源和日期
    4. 不接受DMCA下架请求以外的删除要求

2.9.3 高风险（谨慎考虑，建议规避）
──────────────────────────────────────────────────────────
  数据源                    法律依据              风险说明
  ─────────────────────────────────────────────────────────────────
  Boss直聘/智联招聘/拉勾    反不正当竞争法       中风险。ToS明确禁止爬取。
                                                 司法实践中曾有多起爬取招聘网站被判赔的案例。
                                                 策略: v1.0用公开报告代替，不爬取
  
  百度文库/豆丁网           著作权法             高风险。他人上传内容+平台版权声明。
                                                 策略: 完全不爬取

  具体法律条文参考:
    - 《反不正当竞争法》第12条: 未经同意、未经授权爬取数据可能构成不正当竞争
    - 《网络安全法》第41条: 收集公民个人信息需获得用户同意
    - 《数据安全法》第32条: 任何组织、个人不得窃取或以其他非法方式获取数据
    - 《著作权法》第3条: 文字作品受著作权保护
    - 《民法典》第1034条: 自然人的个人信息受法律保护
  
  司法案例参考（类似爬取案件）:
    - "字节跳动诉微博数据爬取案"（2017）: 爬取公开数据被判构成不正当竞争
    - "百度诉奇虎360robots案"（2013）: 违反robots协议爬取数据被判赔
    - "大众点评诉百度地图案"（2016）: 爬取并使用用户评论被判赔323万
  
  总结: 
  - 省考试院/阳光高考/高校公开报告: 安全，可以直接做
  - 招聘网站: 低风险ToS违规，但司法实践中有被追诉风险，建议先用公开报告替代
  - 百度文库/豆丁网: 高风险，完全不碰
  - 用户自己上传的内容: 安全，用户授权即可（注意隐私数据保护）

2.10.1 数据更新场景
──────────────────────────────────────────────────────────
  场景A: 年度更新（高考数据）
    触发: 每年6月-7月（高考出分+录取结束后）
    数据项: 分数线/一分一段表/招生计划
    操作: 爬虫重新抓取当年数据 → 清洗 → 比对去重 → 追加到数据库
    自动化: 每年6月1日自动触发准备脚本，7月15日全量更新
    告警: 超过7月30日仍未更新成功 → 告警
    
  场景B: 增量更新（院校信息）
    触发: 学校更名/升级/合并时
    数据项: 院校基本信息
    操作: 非定期，手动触发
    自动化: 每月初自动扫描教育部"全国高等学校名单"更新
    
  场景C: 数据修复（质量修复）
    触发: 发现数据错误时
    操作: 手动修正单条或批量修正
    
  场景D: 用户数据（不变更）
    触发: 无（用户自己控制）
    说明: 用户录入的数据属于用户，系统不自动修改

7.3 Prompt工程框架 ★新增
──────────────────────────────────────────────────────────
  每个AI角色由3部分Prompt组成：
  
  【A】角色设定Prompt (System Prompt)
    定义: 身份/目标/约束/输出格式
    示例（高考志愿专家）:
      "你是资深高考志愿填报专家，熟悉中国各省高考政策、
       录取规则和院校情况。你的任务是根据考生的分数、位次、
       兴趣和专业意向，提供科学、客观的志愿填报建议。
       输出格式: JSON，包含冲稳保三档推荐。"
  
  【B】上下文注入Prompt (Context Injection)
    定义: 将用户相关数据注入到Prompt中
    数据来源: 
      - 用户历史考试数据（最近3次考试）
      - 用户目标设定
      - 系统数据库（分数线/位次/院校信息）
    示例:
      "用户最近考试成绩: 数学128/150, 英语135/150, 
       总分580/750。目标院校: 山东大学计算机专业。
       请基于以上数据给出分析。"
  
  【C】输出格式约束Prompt (Output Schema)
    定义: 强制AI输出结构化JSON（便于前端渲染）
    实现: 在System Prompt中指定JSON Schema
  
  Prompt版本管理:
    - 所有Prompt存储在 prompts/ 目录
    - 版本号: v1.0.0, v1.0.1...
    - A/B测试: 可以同时跑两个版本Prompt，比较效果

7.4 AI质量评估框架 ★新增
──────────────────────────────────────────────────────────
  如何衡量AI输出质量？
  
  评估维度:
  1. 准确性: AI建议与真实录取结果的吻合度
  2. 完整性: AI是否覆盖了所有必要分析维度
  3. 可解释性: AI是否给出了清晰的推理过程
  4. 个性化: AI是否基于用户数据给出针对性建议
  
  评估方法:
  - 离线评估: 用2024年真实录取数据测试AI推荐准确性
  - 在线评估: 收集用户反馈（点赞/点踩）
  - 人工评估: 随机抽取10%的AI输出由专家打分
  
  目标指标（v1.0）:
  - 录取概率预测准确率: >85%（AI预测录取概率与实际结果的相关系数）
  - 用户满意度: >4.0/5.0（用户反馈）
  - 推荐覆盖率: 100%（每次查询都给出有效推荐）

12.3 AI幻觉风险控制
──────────────────────────────────────────────────────────
  AI推荐类产品的最大风险是"推荐错误导致用户高考填报失误"。
  虽然我们不是正式的志愿填报服务机构（法律上不需要承担填报失误责任），
  但从用户体验角度，错误推荐会直接杀死产品。

  风险控制措施:
  - 数据约束: AI推荐时参考真实分数线数据库，不允许AI"编造"分数线数据
  - 置信度标注: 每个推荐附带置信度（高/中/低），低置信度标注"建议咨询学校招生办"
  - 数据溯源: 每条推荐数据标注来源（哪一年/哪个省/哪个学校）
  - 人工审核: 每年数据更新时，抽取10%的推荐结果人工核对
  - 用户确认: 每次AI推荐后，用户确认"这个推荐有帮到你吗？" → 数据用于改进

13. 商业模式与盈利规划 ★新增
********************************************************************************

13.0 核心理念
──────────────────────────────────────────────────────────
  "免费吸引→高频使用→自然付费"
  
  核心数据功能免费（考试分析/志愿推荐），增值服务收费。
  不靠广告（不伤害用户体验），不靠卖数据（保护隐私信任）。
  靠"高级AI分析能力"和"数据深度服务"收费。

13.1 定价方案（推荐方案B）
──────────────────────────────────────────────────────────

  ┌─────────────────────────────────────────────────────────┐
  │  方案A: 纯免费（广告模式）                                  │
  │    优点: 用户量最大                                        │
  │    缺点: 一人公司接不到广告，数据不值钱，完全没收入             │
  │    结论: ❌ 不适合                                               │
  ├─────────────────────────────────────────────────────────┤
  │  方案B: 免费增值（推荐 ✅）                                   │
  │    免费版: 基础功能全部免费（考试分析、志愿推荐、目标追踪）       │
  │    付费版: 19元/月 或 199元/年 (选一)                        │
  │    付费解锁: AI深度分析报告、无限次AI咨询、数据导出PDF/PPT     │
  │    结论: ✅ 最适合作一人公司的起步模式                          │
  ├─────────────────────────────────────────────────────────┤
  │  方案C: 付费订阅（无免费版）                                   │
  │    年费: 99元/年                                            │
  │    优点: 直接有收入                                          │
  │    缺点: 没有免费用户=没有传播=0增长                          │
  │    结论: ❌ 没有免费用户就无法增长                                 │
  └─────────────────────────────────────────────────────────┘

  推荐定价: 199元/年（约0.55元/天，对标张雪峰直播5999元的1/30）
  为什么是这个价格?
    - 比夸克VIP(98元)贵一倍 → 因为我们有长期数据积累价值
    - 比张雪峰一对一(5999元)便宜30倍 → 因为我们不是真人，但随时可用
    - 家长一次咨询几百块 → 199元/年完全可接受

13.2 单位经济模型（Unit Economics）
──────────────────────────────────────────────────────────

  【收入端】
  项目                  保守估计      中性估计      乐观估计
  ─────────────────────────────────────────────────────────────
  月活用户(MAU)         1,000         5,000         20,000
  付费转化率            3%            5%            8%
  付费用户数            30            250           1,600
  年费(均值)            199元         199元         199元
  月收入                498元         4,146元       26,533元
  年收入                5,976元       49,750元      318,400元

  注: 一人公司的盈亏平衡点约为月收入15,000元（见成本结构）
  按中性估计: 需要约900个付费用户 → 对应18,000月活 → 估计需要12-18个月达到

  【成本端】
  项目                  月成本(元)    说明
  ────────────────────────────────────────────────────────
  云服务器(阿里云ECS)    50-150       1核2G起步，用户增长后升级
  Claude API调用         50-200      按量付费，取决于用户活跃度
  域名+备案              ~8          阿里云/腾讯云
  代理IP(爬虫用)         0-50         爬取阶段需要，日常不需要
  工具订阅(GitHub/等)    ~20          
  ────────────────────────────────────────
  总月成本              128-428元

  盈亏平衡分析:
  - 固定成本: ~200元/月（服务器+域名+工具）
  - 可变成本: ~50-200元/月（API使用量）
  - 盈亏平衡月收入: ~500元（覆盖成本）
  - 盈亏平衡付费用户: ~30人（收入>成本）
  - 盈利目标月收入: 15,000元（每月养自己）
  - 目标付费用户: ~900人
  
  达到盈亏平衡（月入500元）只需要30个付费用户，这个门槛很低。
  关键是: 如何从0到30个用户？

13.3 用户增长漏斗
──────────────────────────────────────────────────────────

  获客 → 注册 → 活跃 → 付费 → 留存
  
  每个阶段的估算:
  阶段                转化率    1000次曝光带来
  ──────────────────────────────────────────────────
  内容曝光(小红书/知乎)  -       1,000次曝光
  点击到落地页          10%     100次访问
  注册                 20%     20个注册
  周活跃               30%     6个周活跃用户
  付费(3-5%月活)       5%      1个付费用户
  
  例: 发100篇知乎/小红书内容 = 约10万次曝光
  → 1万次网站访问 → 2000注册 → 600周活 → 100付费用户
  → 月收入 = 100 × 199/12 ≈ 1,658元

  用户获取渠道成本对比:
  渠道                成本(元/付费用户)   难度    适合我们?
  ────────────────────────────────────────────────────────
  知乎内容              0(时间成本)      中等     ✅ P0
  小红书内容            0(时间成本)      中等     ✅ P0
  抖音短视频            0(时间成本)      高       ⚠️ P1
  百度搜索竞价          50-200/次点击    高       ❌ 烧不起
  微信朋友圈广告        100+/千次曝光    中       ❌ 烧不起
  与教育机构合作        分佣30-50%       中       ⚠️ P2

  v1.0获客策略:
  渠道1: 知乎——发"高考志愿填报攻略"系列文章，引流到产品
  渠道2: 小红书——发"成绩分析AI"体验内容，吸引高中生和家长
  渠道3: 口碑——通过"考试错题分析"功能自然传播（用户分享成绩图）
